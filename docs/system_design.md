# System Design

## Overview

The system performs **single-step GUI grounding and action prediction** from a screenshot and natural-language instruction. It is NOT a full browser agent — it focuses exclusively on the perception-to-action mapping for one step.

## Pipeline

```
┌─────────────────────────────────────────────────────────────┐

│                    VLM BACKBONE ENCODING                     │

            ┌───────────────────────────────┐
```

## Stage A — Supervised Baseline

- **Input**: (screenshot, instruction) pairs with annotated target elements
- **Training**: Standard cross-entropy / regression loss on ground-truth targets
- **Output**: A fine-tuned VLM that can predict bbox/point + action type
- **Purpose**: Strongest possible supervised baseline before reward optimization

## Stage B — Candidate Generation + Reranking

- **Input**: Each training sample + K candidate actions generated by the SFT model
- **Process**:
  1. Generate candidates via temperature sampling or heuristic construction
  2. Score each candidate with the verifiable reward function
  3. Train a scorer to predict reward from (input, candidate) features
- **Output**: A reranker that selects the best candidate from a set

## Stage C — Policy Improvement

Two options, tried sequentially:

### C-1: Pairwise Preference (DPO-style)
- Construct (preferred, dispreferred) pairs from reward comparisons
- Train the model to increase probability of preferred actions
- Conservative: does not require on-policy sampling at train time

### C-2: Lightweight GRPO
- For each input, generate K candidates as a "group"
- Compute verifiable rewards for each
- Estimate advantages as (reward − group_mean)
- Update policy with clipped gradient to increase above-average candidates
- Operates as a contextual bandit (single-step), not full trajectory RL

## Reward Function Design

The verifiable reward is computed deterministically from ground-truth annotations:

| Component | Signal | Range |
|-----------|--------|-------|
| `element_correct` | Predicted element matches GT | {0, 1} |
| `IoU(b, b*)` | Bounding-box overlap | [0, 1] |
| `click_inside_target` | Click point falls inside GT box | {0, 1} |
| `action_type_correct` | Predicted action matches GT | {0, 1} |
| `invalid_format_penalty` | Malformed output | {0, 1} |

Composite: `r = λ1·elem + λ2·IoU + λ3·click + λ4·act − λ5·penalty`

Default weights: λ1=1.0, λ2=0.5, λ3=0.3, λ4=0.2, λ5=0.5
